# 每日复盘

每天结束时自动执行，从当天的交互中学习并改进。

## 触发条件

由 Heartbeat 服务在每天固定时间触发（建议 21:00）。

## 执行步骤

### 1. 收集当日数据

- 使用 `session_manager` 查询今天的会话记录
- 统计：消息数、工具调用次数、错误次数
- 提取：使用的工具列表、用户请求类型

### 2. 分析当日交互

分析以下维度：
- **成功模式**：哪些类型的请求处理得最好？
- **失败教训**：哪些工具调用失败了？为什么？
- **重复操作**：哪些任务需要重复手动执行？
- **用户需求**：用户最常问什么？
- **新技能需求**：有哪些可以自动化的任务？

### 3. 决策输出

根据分析结果决定：

| 情况 | 操作 |
|------|------|
| 简单经验/用户偏好 | 更新 `memory/MEMORY_GRAPH.md` |
| 复杂/可复用模式 | 创建新 skill 到 `workspace/skills/` |
| 需要新工具 | 创建新 tool 到 `workspace/tools/` |
| 重要进展 | 记录到 `memory/TIMELINE.md` |

### 4. 生成复盘报告

输出格式：
```
## 每日复盘 {日期}

### 统计
- 消息数: X
- 工具调用: Y 次
- 错误: Z 次

### 学到的经验
- ...

### 创建的技能/工具
- ...

### 下次改进
- ...
```

使用 `message` 工具将复盘报告发送给用户。

## 质量原则

1. **每天只创建1个新技能**：避免过度工程化
2. **优先复用现有技能**：先检查现有 skills
3. **保持技能小而专注**：一个技能解决一个问题
4. **验证后再创建**：确保新技能真正有用

## 禁止事项

- 不要修改 builtin skills (`nanobot/skills/`)
- 不要创建过于宽泛的技能
- 不要在复盘报告中包含敏感信息
